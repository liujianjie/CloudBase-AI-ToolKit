# Tag 体系设计方案

## 一、Tag 维度设计

### 1. 终端类型（Platform）
用于区分应用运行的平台/终端

- **小程序** (miniprogram) - 微信小程序相关
- **H5** (h5) - H5 移动端应用
- **Web** (web) - Web 网页应用
- **全栈** (fullstack) - 包含前后端的完整应用

### 2. 应用类型（Category）
用于区分应用的功能类型

- **游戏** (game) - 各类游戏应用
- **工具** (tool) - 实用工具类应用
- **社交** (social) - 社交类应用
- **电商** (ecommerce) - 电商购物类应用
- **教育** (education) - 教育学习类应用
- **管理系统** (management) - 后台管理系统、排班系统等
- **其他** (other) - 其他类型应用

### 3. 开发工具（IDE）
用于区分使用的开发工具/IDE

- **Cursor** (cursor) - Cursor IDE
- **CodeBuddy** (codebuddy) - CodeBuddy IDE
- **Claude Code** (claude-code) - Claude Code
- **其他** (other-ide) - 其他开发工具

## 二、Tag 使用规则

### 视频 Tag 规则
- 每个视频可以有一个或多个 tag
- 必须至少包含一个"终端类型" tag
- 可以包含多个"应用类型" tag（如果涉及多个类型）
- 可以包含一个"开发工具" tag（如果明确提到）

### 文章 Tag 规则
- 每个文章可以有一个或多个 tag
- 必须至少包含一个"终端类型" tag
- 可以包含多个"应用类型" tag
- 可以包含一个"开发工具" tag

## 三、Tag 展示方式

### 筛选器设计
- 在视频/文章列表上方显示筛选器
- 支持多维度筛选（终端类型、应用类型、开发工具）
- 支持多选筛选
- 显示当前筛选结果数量
- 提供"清除筛选"功能

### Tag 标签展示
- 在每个视频/文章卡片上显示相关 tag
- Tag 使用不同颜色区分维度
- 点击 tag 可快速筛选
